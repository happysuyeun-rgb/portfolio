Process — 8 Steps
01
Use Case 정리 · 버전 관리
기획 시작점
서비스가 해결할 문제와 사용자 행동 시나리오를 정의합니다. 변경 이력을 버전으로 관리해 의사결정 근거를 보존합니다.
사용자 정의 시나리오 버전 관리 Notion
02
IA 구조 설계
정보 설계
전체 서비스의 정보 구조와 탐색 흐름을 설계합니다. 권한 체계와 RLS 방향도 이 단계에서 초안을 잡습니다.
IA 설계 탐색 구조 권한 체계 초안 Figma
2-1
기능정의서 · 요구사항 정리
선행 문서
화면 설계 전에 기능 목록과 요구사항을 먼저 확정합니다. 화면이 기능을 따르는 구조를 만듭니다.
기능 목록 요구사항 Confluence Jira
2-2
화면정의서 · 에러케이스 정의
설계 문서
화면 목록과 흐름을 정의합니다. 에러케이스·예외 처리는 이 단계에서 함께 정의해 후반 수정 비용을 줄입니다.
화면 목록 에러케이스 예외 처리 Notion
03
✦ 핵심 차별점 — 대부분의 기획자가 하지 않는 단계
Cursor AI 목업 구현 · 배포 · 시연
AI 구현
Cursor AI로 실제 동작하는 프로토타입을 직접 구현하고 배포합니다. 기획 가설을 코드로 검증한 뒤 개발팀에 레퍼런스로 전달합니다. 기획서와 구현 사이의 간극을 제거합니다.
Cursor AI React Next.js Supabase Vercel 배포 PWA GitHub 시연
04
화면설계서 완료 · 개발자 점검
협업
화면설계서를 완성하고 개발자와 점검합니다. 동작하는 레퍼런스를 기준으로 수정·보완 사항을 구체적으로 논의합니다.
화면설계서 개발자 점검 수정 보완 Slack Jira
05
Figma UI/UX · 프로토타입 · 시연
디자인
Figma에서 컴포넌트 상태 정의, 디자인 시스템 구축, 인터랙션 프로토타입을 작업합니다. 완성 후 팀에 시연하고 공유합니다.
Figma 컴포넌트 시스템 디자인 시스템 WCAG 접근성 프로토타입
06
에러케이스 분기 처리
검증
전체 흐름에서 누락된 에러케이스를 최종 검토하고 분기 처리를 확정합니다. 앞 단계에서 정의한 내용을 기준으로 점검합니다.
에러 분기 예외 시나리오 최종 검토
07
RLS 정책 수립 · 권한별 정책 확정
보안 · 정책
사용자 권한별 데이터 접근 정책을 확정합니다. Row Level Security 설계를 통해 보안과 UX 흐름이 일치하는지 검증합니다.
RLS 권한 정책 보안 설계 Supabase
08
최종 문서 정리
마무리
전 과정의 산출물을 정리하고 버전을 확정합니다. 개발팀 인계 문서와 히스토리를 보존합니다.
산출물 정리 버전 확정 인계 문서 Notion Confluence
