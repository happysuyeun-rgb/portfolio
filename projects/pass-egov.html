모바일 신뢰 기반의 공공 고지 UX

PASS 지갑 내 전자고지 서비스

본 프로젝트에서 역할은 UI/UX 총괄이었습니다.

#UX Strategy #UI/UX #Design System

Project Overview

  • PASS 앱 내에서 각종 고지서(건강보험, 세금, 범칙금 등)의 수신·열람·결제까지 연결하는 풀 플로우 설계
  • 민간 수준의 편의성과 공공 보안 기준을 동시에 충족하는 UX 가이드 수립
  • 배지/이력서 등 고용24 연계 시나리오를 고려해 모듈형 컴포넌트화
  • 툴: Figma, FigJam, Jira, Confluence

Problem

  • 기관별 서로 다른 문구·양식으로 인해 사용자가 고지 흐름을 파악하기 어려움
  • 결제 이전 단계에서 고지서 진위 확인/상세 비교가 충분히 제공되지 않음
  • 접근성(명도, 포커스 이동, 스크린리더 레이블)이 고지 플로우마다 상이

Outcome

IA / 검색 UX

  • 의도 기반 자동완성/추천으로 검색 → 고지서 열람까지 클릭 최소화
  • 핵심 메타정보(발행기관, 납부기한, 금액)를 상단 카드로 요약
  • 상세 비교 보기를 고정 패널로 제공해 이탈 없이 확인 가능

시각체계 / 확장성

  • 브랜드·기관 로고가 혼재해도 일관되게 보이는 컴포운드 토큰 설계
  • 다크모드 및 다양한 고지서 템플릿 확장 대응
  • 결제/영수증/이력 연계까지 재사용 가능한 UI 패턴 구축

Reflection

  • ‘빠른 납부’보다 ‘안심하고 납부’에 초점을 둔 내러티브가 전환율에 더 긍정적
  • 기관별 사소한 예외가 실제 오류로 이어지기 쉬워 QA 플로우를 강화
  • 설계-디자인-가이드를 한 소스에서 관리해 협업 효율 높임

Impact & Value

  • 고지서 열람→결제 전환율 상승(추정)과 민원 상담건수 감소에 기여
  • PASS 내 다른 서비스에서도 재사용 가능한 컴포넌트로 운영 효율↑
  • 공공 고지 UX의 참조 사례로 활용
