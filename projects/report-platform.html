시민 참여형 모바일 플랫폼 · 제보서비스 · 브랜딩 + UX/UI 총괄

시민이 현장에서 직접 안전·편의 정보를 제보하고,
검증을 거쳐 지역사회와 공유되는 참여형 플랫폼 —
브랜드 아이덴티티 정의부터 제보 플로우,
신뢰 기반 커뮤니티 UX 설계까지 총괄했습니다.

Project Overview

Role 브랜딩 · UX/UI 디자인 총괄
Platform iOS / Android 모바일 앱
Scope 제보 플로우 · 지도 UI · 커뮤니티 · 관리자 화면
Tools Figma · Map SDK · Amplitude 이벤트 설계

The Challenge

배경 및 문제점

  • 제보가 지도 위에 흩어져 맥락 파악이 어렵고 중복·오정보 제보가 섞이는 신뢰 문제
  • 가짜·홍보성 제보를 거르는 검증 기준과 절차가 불명확해 운영 부담 가중
  • 현장에서 제보하는 모바일 특성상 입력 피로·이미지 용량·위치 정확도 문제로 완료율이 낮음

The Solution

핵심 해결 방향

  • 브랜드 아이덴티티 구축 — 시민 참여와 신뢰를 시각적으로 표현하는 브랜드 비주얼 시스템과 디자인 가이드 수립
  • 트러스트 디자인 — 제보→검증→공유의 전 과정을 투명하게 보여주는 상태 시각화 UX로 플랫폼 신뢰도 구축
  • 모바일 최적화 제보 플로우 — 스텝폼·위치 자동보정·중복 감지로 현장 입력 완료율을 높이고 데이터 품질 확보

디자인 의사결정 — Key Design Decisions

01

현장 입력을 위한 스텝폼 설계

시민이 사고 현장·도로·시설물 앞에서 스마트폰으로 제보하는 상황을 전제로 설계했습니다. 한 화면에 모든 정보를 입력하는 방식 대신, 카테고리 선택→위치 확인→사진 첨부→내용 입력으로 단계를 분리해 입력 피로를 낮추고 완료율을 높였습니다. 위치는 GPS 자동 보정으로 오입력을 최소화했습니다.

02

중복 제보 감지 UX로 데이터 품질 확보

같은 지점에 중복 제보가 쌓이면 운영 효율이 떨어지고 지도가 혼잡해집니다. 제보 입력 시 반경 내 유사 제보를 자동 감지해 "이미 접수된 제보가 있습니다"를 인라인으로 안내하고, 기존 제보에 공감하거나 추가 정보를 보완하는 방향으로 유도했습니다. 차단이 아닌 '합류'로 설계한 것이 핵심이었습니다.

03

게이미피케이션으로 지속 참여 유도

제보 후 결과를 알 수 없으면 참여 동기가 사라집니다. 제보 처리 상태를 대기→검증중→처리완료 타임라인으로 시각화하고, 누적 기여도에 따라 시민 배지와 레벨을 부여하는 게이미피케이션 구조를 설계했습니다. 개인의 기여가 지역사회에 실제 변화를 만든다는 피드백 루프를 UX로 구현했습니다.

Step 01

제보 플로우 — 현장 최적화 스텝폼

복잡한 입력을 카테고리→위치→사진→내용의 4단계 스텝폼으로 분리했습니다. GPS 위치 자동 보정과 이미지 압축 처리로 현장에서 누구나 정확하게 제보할 수 있는 모바일 입력 UX를 구현했습니다.

Step 02

신뢰 — 중복 감지 & 검증 UX

제보 입력 시 반경 내 유사 제보를 자동 감지해 인라인으로 안내합니다. 중복 차단이 아닌 기존 제보에 공감하거나 정보를 보완하는 방향으로 유도해 데이터 품질과 커뮤니티 참여를 동시에 높였습니다.

Step 03

참여 — 상태 타임라인 & 시민 배지

제보 접수부터 해결까지 처리 상태를 타임라인으로 시각화하고, 누적 기여도에 따라 시민 배지·레벨을 부여하는 게이미피케이션 구조를 설계했습니다. 개인의 제보가 지역사회 변화로 이어지는 피드백 루프를 UX로 구현했습니다.

Impact & 성과

유효 제보율 상승

스텝폼·중복 감지 UX 도입으로 유효 제보율 대폭 상승. 지자체 협업 파일럿에서 사용자 참여와 데이터 품질 동시 향상.

시민 참여형 문제 해결 모델

개인 제보 → 지역사회 공유로 이어지는 참여형 플랫폼의 참고 사례 확립.

운영 데이터 기반 품질 개선

맵·카드·포털 컴포넌트 재사용으로 유지보수 효율 향상. 운영 데이터 기반 품질 개선 선순환 구조 구현.
