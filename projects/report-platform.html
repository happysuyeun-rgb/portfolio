참여형 플랫폼의 경험 디자인

위치기반 정보 공유 플랫폼 – 제보서비스

본 프로젝트에서 역할은 Product Design 총괄이었습니다.

#UX Strategy #System Architecture #Product Design #Design System

Project Overview

  • 시민이 위치 기반으로 생활 안전/편의 정보를 제보·공유하는 플랫폼
  • 제보→검증→공유의 전 과정을 투명하게 보여주는 트러스트 디자인
  • 커뮤니티 참여 유인을 위한 배지/레벨/기여도 피드백 체계
  • 툴: Figma, Map SDK, Amplitude 이벤트 설계

Problem

  • 제보가 지도에 흩어져 맥락 파악이 어려움(중복/오정보 이슈)
  • 가짜/홍보성 제보를 거르는 기준과 절차가 불명확
  • 모바일 사용성(입력 피로, 이미지 용량, 위치 정확도) 이슈

Outcome

IA / 검색 UX

  • 제보 카테고리와 위치 정확도를 일차 검증하는 ‘스텝폼’
  • 중복 제보 자동 감지 및 유사 항목 제안
  • 검색/필터/타임라인을 결합한 탐색 흐름

시각체계 / 확장성

  • 상태 뱃지(대기/검증중/공유중)와 기여도 메트릭 시각화
  • 지도/리스트/카드 3뷰의 디자인 토큰 통합
  • 관리 포털에서 정책을 바꿔도 사용자 UI가 자동 반영되도록 스키마화

Reflection

  • ‘빠른 제보’보다 ‘신뢰할 수 있는 제보’가 재방문에 영향
  • 사진/위치 권한 안내를 마찰 없는 마이크로카피로 해결
  • 운영 데이터로 품질 지표를 대시보드화해 개선 루프 구축

Impact & Value

  • 지자체 협업 시 파일럿에서 사용자 참여와 유효 제보율 상승
  • 도시 문제 해결형 크라우드소싱 모델의 참고 사례
  • 맵/카드/포털 컴포넌트 재사용으로 유지보수 비용 절감
